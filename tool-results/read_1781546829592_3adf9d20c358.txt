  1375‚Üí  1375‚ÜíAuthentication Security
  1376‚Üí  1376‚ÜíRequirement Implementation
  1377‚Üí  1377‚ÜíPasswords hashed Supabase Auth (bcrypt, 10 rounds in auth.users)
  1378‚Üí  1378‚ÜíNo password_hash in customers table Supabase Auth is sole owner of passwords
  1379‚Üí  1379‚ÜíJWT storage HTTP-only cookie, Secure, SameSite=Lax
  1380‚Üí  1380‚ÜíCustomer session 7 days (Supabase JWT expiry)
  1381‚Üí  1381‚ÜíAdmin session Manual logout only ‚ÄĒ no idle timeout
  1382‚Üí  1382‚ÜíPassword policy Admin: 16+ chars, unique; enforced in seed script
  1383‚Üí  1383‚ÜíInitial admin Seed script only; no public admin registration route
  1384‚Üí  1384‚ÜíRate Limiting (Database-based - No Redis, No Upstash)
  1385‚Üí  1385‚ÜíEndpoint Limit Window Storage Notes
  1386‚Üí  1386‚ÜíLogin 5 attempts 15 min customers.login_attempts / locked_until Per email
  1387‚Üí  1387‚ÜíOrder tracking 10 attempts 15 min order_track_attempts Per IP; rolling window
  1388‚Üí  1388‚ÜíReview submission 5 submissions 1 hour Application logic Per customer
  1389‚Üí  1389‚ÜíThere is no Redis, Upstash, or any external caching service in V1. All rate limiting uses Supabase PostgreSQL.
  1390‚Üí  1390‚Üí
  1391‚Üí  1391‚Üí--- PAGE 41 ---
  1392‚Üí  1392‚ÜíRow Level Security (RLS)
  1393‚Üí  1393‚ÜíÔā∑ Principle: Least privilege - every table has RLS enabled.
  1394‚Üí  1394‚ÜíÔā∑ Service role bypasses RLS for admin server-side operations only.
  1395‚Üí  1395‚ÜíCritical: Guest INSERT policies - The following tables must have INSERT policies that allow unauthenticated users for guest checkout to
  1396‚Üí  1396‚Üíwork:
  1397‚Üí  1397‚ÜíÔā∑ customers - allow INSERT for guest creation
  1398‚Üí  1398‚ÜíÔā∑ orders - allow INSERT for guest order creation
  1399‚Üí  1399‚ÜíÔā∑ order_items - allow INSERT for order item creation
  1400‚Üí  1400‚ÜíÔā∑ order_track_attempts - allow INSERT for rate limiting
  1401‚Üí  1401‚ÜíWithout these INSERT policies, guest checkout will fail with RLS violations.
  1402‚Üí  1402‚ÜíAdmin Route Protection (Middleware)
  1403‚Üí  1403‚ÜíÔā∑ Checks role in customers table on every /admin/* request.
  1404‚Üí  1404‚ÜíÔā∑ Redirects to /login if not authenticated or not admin.
